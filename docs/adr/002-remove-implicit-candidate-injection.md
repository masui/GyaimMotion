# ADR-002: 暗黙的な候補挿入機能の削除

## Status

Accepted

## Decision

以下の3つの暗黙的な候補挿入機能を削除する。

1. クリップボード内容の候補先頭挿入（直近5秒以内のコピー内容）
2. 選択テキストの候補先頭挿入 + 確定時の辞書自動登録
3. `?` トリガーによる MD5 XOR 暗号化/復号

## Context

- クリップボード候補: 意図しないタイミングで候補に混入し、ユーザーが意図した変換を妨げる
- 選択テキスト候補: `selectedStr` が一度セットされるとリセットされず、無関係な入力時にも候補に出続けるバグがある
- 暗号機能: `?` が通常の文字として入力できなくなる。全角`？`の入力にも支障が出る

## Consideration

- いずれの機能も「叶えたい目的」自体は妥当
- しかし暗黙的にトリガーされる設計がユーザー体験を損なっている
- 明示的な操作（専用UI、特定キー操作）で同等の機能を提供する方が適切

## Consequences

- `?` `.` `!` 等の記号が通常の入力文字として使えるようになる
- rklistに全角マッピング (`?`->`？` 等) を追加し、全角記号入力が可能に
- 辞書登録は DictEditorWindow (GUI) で明示的に行う運用に変更
- 将来の再実装は Issue #2 で追跡
